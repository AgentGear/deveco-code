# System Kit API Quick Reference

Verified import + call shapes for the system kit APIs that AI most often hallucinates. UI components live in the cookbook; this file covers the `@kit.*` namespaces behind them. **Any namespace member not shown here is unverified — look it up in the SDK `.d.ts`, never guess** (kit namespaces are where "plausible" names fail most: every entry in the fabricated table below was generated by a model and rejected by the checker).

> Shapes verified against OpenHarmony API 12+ kit exports. All examples are ArkTS-strict: no destructuring, no spread on non-arrays, `BusinessError` in catch.

## Fabricated kit members (observed in production — none of these exist)

| ❌ Generated | ✅ Reality |
|------------|-----------|
| `fileIo.WriteOptions` / `fileIo.ReadOptions` type annotations | not exported through the `fileIo` namespace — use the option-free overloads below; for whole-file text use `readTextSync` |
| `abilityAccessCtrl.PermissionRequestResult` | `PermissionRequestResult` is a TOP-LEVEL export of `@kit.AbilityKit`, not a member of the namespace |
| `abilityAccessCtrl.AccessControlManager` | not an exported type — `createAtManager()` returns an opaque handle; let the type be inferred: `const atManager = abilityAccessCtrl.createAtManager();` |
| `import { audio } from '@kit.MediaKit'` | `audio` namespace lives in **`@kit.AudioKit`**, not `@kit.MediaKit`. `@kit.MediaKit` exports `media` (AVPlayer / AVRecorder) |
| `photoAccessHelper.CreateRequest` | save via `picker.DocumentViewPicker().save(...)` or `SaveButton`; asset writes go through `photoAccessHelper.MediaAssetChangeRequest` |
| `import { picker } from '@kit.MediaLibraryKit'` / `'@kit.ArkUI'` | the photo/document `picker` namespace is exported by `@kit.CoreFileKit` |
| `PhotoSelectOptions.maxSelectNum` | `maxSelectNumber` |
| `pixelMap.filter(...)`, `image.PixelMapFilterBlurType` | `PixelMap` has NO filter/blur methods — blur the rendered `Image` component with the `.blur(radius)` modifier |
| `context.getEventHub()` | `eventHub` is a PROPERTY: `context.eventHub.emit(...)` |
| `Preferences.getString(key)` / `.putString(key, val)` / `.getInt()` / `.putInt()` | 不存在；统一用 `getSync(key, default)` / `putSync(key, value)` |
| `promptAction.ToastOptions` / `ToastOptions` type annotation | not an exported type — pass a plain object literal `{ message: '...' }` to `showToast()` directly; for typed context use `import { promptAction } from '@kit.ArkUI'` and call `promptAction.showToast({ message: '...' })` |

## 1. File read/write (`fileIo`)

```ets
import { fileIo as fs } from '@kit.CoreFileKit';
import { common } from '@kit.AbilityKit';

// App-private dirs need no permission
const context = getContext(this) as common.UIAbilityContext;
const path = `${context.filesDir}/notes.json`;

// Write text (CREATE + TRUNC for overwrite, APPEND to append)
const file = fs.openSync(path, fs.OpenMode.READ_WRITE | fs.OpenMode.CREATE | fs.OpenMode.TRUNC);
fs.writeSync(file.fd, JSON.stringify(data));
fs.closeSync(file);

// Read whole text file — one call, no buffers, no options types
if (fs.accessSync(path)) {
  const content: string = fs.readTextSync(path);
}
```

- Do NOT annotate read/write options with `fileIo.ReadOptions` / `fileIo.WriteOptions` — the simple overloads above cover app use cases without naming an options type.
- Binary data: `fs.readSync(file.fd, arrayBuffer)` / `fs.writeSync(file.fd, arrayBuffer)` — pass the buffer positionally, no options literal.

## 2. Runtime permission request (`user_grant`)

```ets
import { abilityAccessCtrl, Permissions, PermissionRequestResult, common } from '@kit.AbilityKit';
import { BusinessError } from '@kit.BasicServicesKit';

const PERMS: Array<Permissions> = ['ohos.permission.MICROPHONE'];

async requestMic(): Promise<boolean> {
  const context = getContext(this) as common.UIAbilityContext;
  const atManager = abilityAccessCtrl.createAtManager();
  try {
    const result: PermissionRequestResult = await atManager.requestPermissionsFromUser(context, PERMS);
    // authResults: 0 = granted, -1 = denied — one entry per requested permission
    return result.authResults.length > 0 && result.authResults[0] === 0;
  } catch (e) {
    console.error(`permission request failed: ${(e as BusinessError).code}`);
    return false;
  }
}
```

- `PermissionRequestResult` and `Permissions` are top-level `@kit.AbilityKit` exports — NOT members of `abilityAccessCtrl`.
- Every `user_grant` permission must also be declared in `module.json5` with `reason` + `usedScene` (see SKILL.md change-set checks).

## 3. Photo picker (select images — no permission needed)

```ets
import { picker } from '@kit.CoreFileKit';

async pickImage(): Promise<string> {
  const options = new picker.PhotoSelectOptions();
  options.MIMEType = picker.PhotoViewMIMETypes.IMAGE_TYPE;
  options.maxSelectNumber = 1;                       // NOT maxSelectNum
  const photoPicker = new picker.PhotoViewPicker();
  const result: picker.PhotoSelectResult = await photoPicker.select(options);
  return result.photoUris.length > 0 ? result.photoUris[0] : '';
}
```

- `picker` comes from `@kit.CoreFileKit` — not from `@kit.MediaLibraryKit` and not from `@kit.ArkUI`.
- Using the picker requires NO media permissions; only direct media-store access does.

## 4. Saving a file the user can see (document picker save)

```ets
import { picker, fileIo as fs } from '@kit.CoreFileKit';

async exportText(content: string): Promise<string> {
  const options = new picker.DocumentSaveOptions();
  options.newFileNames = ['export.txt'];
  const docPicker = new picker.DocumentViewPicker();
  const uris: Array<string> = await docPicker.save(options);
  if (uris.length === 0) { return ''; }
  const file = fs.openSync(uris[0], fs.OpenMode.READ_WRITE | fs.OpenMode.CREATE);
  fs.writeSync(file.fd, content);
  fs.closeSync(file);
  return uris[0];
}
```

- Do not invent `photoAccessHelper.CreateRequest` — saving into the photo album needs `photoAccessHelper.MediaAssetChangeRequest` + `applyChanges`, or the `SaveButton` security component; for plain files prefer the document picker above.

## 5. Image processing (`image` kit, PixelMap)

```ets
import { image } from '@kit.ImageKit';

const source: image.ImageSource = image.createImageSource(filePath);
const pm: image.PixelMap = await source.createPixelMap();

await pm.scale(0.5, 0.5);
await pm.crop({ x: 0, y: 0, size: { width: 100, height: 100 } });
await pm.rotate(90);
pm.release();
```

- `PixelMap` supports `scale` / `crop` / `rotate` / `flip` / `opacity` — there is NO `filter`, NO blur, NO color-matrix API on it. Visual blur is a UI concern: `Image(this.pm).blur(20)`.
- Encode with `image.createImagePacker()` + `packing(pm, { format: 'image/jpeg', quality: 90 } as image.PackingOption)`.

## 6. eventHub (in-app pub/sub)

```ets
import { common } from '@kit.AbilityKit';

const context = getContext(this) as common.UIAbilityContext;
context.eventHub.emit('cartChanged', this.cartCount);          // property, not getEventHub()
context.eventHub.on('cartChanged', (count: number) => { this.badge = count; });
context.eventHub.off('cartChanged');                           // clean up in aboutToDisappear
```

- For parent↔child component communication prefer `@Link`/`@Prop`/callbacks; eventHub is for cross-page/ability signals.

## 7. Media playback (`media` kit, AVPlayer)

```ets
import { media } from '@kit.MediaKit';
import { BusinessError } from '@kit.BasicServicesKit';

const avPlayer: media.AVPlayer = await media.createAVPlayer();
avPlayer.url = 'file:///data/storage/...';  // 本地文件用 url 字符串
// 或 fd 模式:
// avPlayer.fdSrc = { fd: file.fd, offset: 0, length: fileSize } as media.AVFileDescriptor;

avPlayer.on('stateChange', (state: string) => { /* idle/initialized/prepared/playing/paused/stopped/released */ });
avPlayer.on('timeUpdate', (time: number) => { /* 播放进度 ms */ });
await avPlayer.prepare();
await avPlayer.play();
```

- `AVPlayer` 数据源: `url` (string) 或 `fdSrc` (`media.AVFileDescriptor`) — 没有 `AvPlayerFdSrc` 类型
- `AudioViewPicker` 的 `.select()` 返回 `Promise<Array<string>>` — 没有 `AudioSelectResult` 类型
- Slider 控制播放进度时，`.onChange((value: number, mode: SliderChangeMode)` — `SliderChangeMode` 成员为 `Begin`/`Moving`/`End`/`Click`（没有 `Flick`）

## 8. Audio recording (`audio` kit, AudioCapturer)

```ets
import { audio } from '@kit.AudioKit';          // NOT @kit.MediaKit
import { fileIo as fs } from '@kit.CoreFileKit';
import { BusinessError } from '@kit.BasicServicesKit';

// AudioCapturer — record PCM to file
const options: audio.AudioCapturerOptions = {
  streamInfo: {
    samplingRate: audio.AudioSamplingRate.SAMPLE_RATE_44100,
    channels: audio.AudioChannel.CHANNEL_1,
    sampleFormat: audio.AudioSampleFormat.SAMPLE_FORMAT_S16LE,
    encodingType: audio.AudioEncodingType.ENCODING_TYPE_RAW,
  },
  capturerInfo: {
    source: audio.SourceType.SOURCE_TYPE_MIC,
    capturerFlags: 0,
  },
};
const capturer: audio.AudioCapturer = await audio.createAudioCapturer(options);

const path = `${getContext(this).filesDir}/record.pcm`;
const file = fs.openSync(path, fs.OpenMode.READ_WRITE | fs.OpenMode.CREATE | fs.OpenMode.TRUNC);

capturer.on('readData', (buffer: ArrayBuffer) => {
  fs.writeSync(file.fd, buffer);     // sequential write, no options type needed
});

await capturer.start();
// ... later:
await capturer.stop();
fs.closeSync(file);
capturer.release();
```

- `audio` comes from **`@kit.AudioKit`** — `@kit.MediaKit` exports `media` (AVPlayer/AVRecorder), NOT `audio`
- Do NOT annotate options as `fileIo.WriteOptions` / `fileIo.ReadOptions` — use sequential positional writes/reads
- Do NOT annotate `abilityAccessCtrl.AccessControlManager` — let `createAtManager()` infer its return type
- MICROPHONE permission is `user_grant`: must declare in `module.json5` with `reason` + `usedScene`

## 9. Canvas game loop pattern (游戏/动画类应用)

Use this pattern for games, animations, or any interactive Canvas rendering. Do NOT switch between Canvas and component-based approaches — pick Canvas and commit.

```ets
import { display } from '@kit.ArkUI';

@Entry
@Component
struct GamePage {
  private context: CanvasRenderingContext2D =
    new CanvasRenderingContext2D(new RenderingContextSettings(true));
  @State canvasWidth: number = 360;
  @State canvasHeight: number = 640;
  private timerId: number = -1;

  aboutToAppear(): void {
    // Get screen size in vp — no deprecated vp2px needed
    const screenInfo: display.Display = display.getDefaultDisplaySync();
    // display returns px; convert to vp: vp = px / densityPixels
    this.canvasWidth = screenInfo.width / screenInfo.densityPixels;
    this.canvasHeight = screenInfo.height / screenInfo.densityPixels;
  }

  aboutToDisappear(): void {
    clearInterval(this.timerId);
  }

  build() {
    Stack() {
      Canvas(this.context)
        .width('100%')
        .height('100%')
        .onReady(() => {
          this.timerId = setInterval(() => { this.gameLoop(); }, 33);
        })
        .onTouch((event: TouchEvent) => {
          // event.touches[0].x / .y are in vp, matching component coordinates
          if (event.type === TouchType.Down || event.type === TouchType.Move) {
            const touch: TouchObject = event.touches[0];
            this.handleTouch(touch.x, touch.y);
          }
        })
    }
  }

  gameLoop(): void {
    const ctx = this.context;
    ctx.clearRect(0, 0, this.canvasWidth, this.canvasHeight);
    // draw game objects using ctx.fillRect / ctx.arc / ctx.fillText / ctx.drawImage ...
  }

  handleTouch(x: number, y: number): void {
    // collision detection in vp coordinates
  }
}
```

### Game object modeling (copy this pattern — do NOT invent class shapes)

```ets
// Separate file: game/GameModel.ets
// All fields initialized — avoids object-literal type-context traps
export class Sprite {
  x: number = 0;
  y: number = 0;
  vx: number = 0;
  vy: number = 0;
  radius: number = 20;
  active: boolean = true;

  static create(x: number, y: number, vx: number, vy: number): Sprite {
    const s = new Sprite();
    s.x = x; s.y = y; s.vx = vx; s.vy = vy;
    return s;
  }
}
```

### Physics update + lifecycle (standard gravity / bounds / filter)

```ets
private sprites: Sprite[] = [];
private gravity: number = 0.3;

updatePhysics(): void {
  for (let i = 0; i < this.sprites.length; i++) {
    const s: Sprite = this.sprites[i];
    s.x += s.vx;
    s.y += s.vy;
    s.vy += this.gravity;
    if (s.y > this.canvasHeight + 50) {
      s.active = false;
    }
  }
  this.sprites = this.sprites.filter((s: Sprite): boolean => s.active);
}
```

### Collision detection (distance-based)

```ets
private hitTest(x: number, y: number, s: Sprite): boolean {
  const dx: number = x - s.x;
  const dy: number = y - s.y;
  return Math.sqrt(dx * dx + dy * dy) < s.radius;
}
```

### Game state machine

```ets
@State gameState: string = 'start';  // 'start' | 'playing' | 'gameover'

// Touch handler dispatches by state — no enum needed
private handleTouchEvent(event: TouchEvent): void {
  if (this.gameState === 'start' && event.type === TouchType.Down) {
    this.gameState = 'playing';
    this.resetGameData();
    return;
  }
  if (this.gameState === 'gameover' && event.type === TouchType.Down) {
    this.gameState = 'playing';
    this.resetGameData();
    return;
  }
  if (this.gameState === 'playing') {
    if (event.type === TouchType.Down || event.type === TouchType.Move) {
      const touch: TouchObject = event.touches[0];
      this.handleTouch(touch.x, touch.y);
    }
  }
}
```

### Spawning pattern (random position + upward velocity)

```ets
private spawnSprite(): void {
  const x: number = Math.random() * this.canvasWidth;
  const vx: number = (Math.random() - 0.5) * 4;
  const vy: number = -(8 + Math.random() * 4);
  this.sprites.push(Sprite.create(x, this.canvasHeight + 30, vx, vy));
}
```

Key decisions (do NOT re-derive these):
- **`display` is in `@kit.ArkUI`** — `display.getDefaultDisplaySync()` returns `Display` with `width`/`height` (px) and `densityPixels`
- **`TouchEvent.touches[0].x/.y` are in vp** — same coordinate space as component layout, no conversion needed
- **Game loop: `setInterval` + `clearInterval`** — `animateTo` is for UI property transitions, not frame-by-frame rendering
- **Canvas sizing**: set Canvas to `100%` and use `this.canvasWidth/Height` (vp) for draw coordinates
- **`RenderingContextSettings`** — NOT `CanvasRenderingContext2DSettings` (does not exist)
- **No `@Observed`/`@ObjectLink` needed** — Canvas redraws imperatively, not via state-driven re-render
- For score/UI overlays, use `Stack` with `Text` components on top of Canvas, not Canvas text rendering

Canvas drawing cheat sheet (do NOT search for these — they are standard):
- `ctx.fillStyle = '#ff0000'` → set before `fillRect(x, y, w, h)` / `fill()`
- Circle: `ctx.beginPath()` → `ctx.arc(x, y, r, 0, Math.PI * 2)` → `ctx.fill()`
- Rotated sprite: `ctx.save()` → `ctx.translate(x, y)` → `ctx.rotate(angle)` → draw at origin → `ctx.restore()`
- Text on Canvas: `ctx.font = '24vp sans-serif'` → `ctx.fillText('Score: 10', x, y)`
- Fade: `ctx.globalAlpha = 0.5` → draw → `ctx.globalAlpha = 1.0`

## 9. Preferences (轻量 KV 存储)

```ets
import { preferences } from '@kit.ArkData';
import { common } from '@kit.AbilityKit';

const context = getContext(this) as common.UIAbilityContext;
const store: preferences.Preferences = preferences.getPreferencesSync(context, { name: 'settings' });

// 读取 — getSync 指定默认值，返回 ValueType（再窄化到具体类型）
const theme: string = store.getSync('theme', 'light') as string;
const volume: number = store.getSync('volume', 50) as number;
const enabled: boolean = store.getSync('enabled', true) as boolean;

// 写入 — putSync + flush
store.putSync('theme', 'dark');
store.putSync('volume', 80);
await store.flush();
```

- **没有** `getString` / `putString` / `getInt` / `putInt` — 只有泛型 `getSync(key, defaultValue)` 和 `putSync(key, value)`
- `getSync` 返回 `preferences.ValueType`（= `number | string | boolean | Array<number> | Array<string> | Array<boolean> | Uint8Array | object`），需要 `as string` 等窄化
- 异步版本: `get(key, defaultValue)` / `put(key, value)` 返回 Promise
- 删除: `store.deleteSync(key)` / `store.delete(key)`
